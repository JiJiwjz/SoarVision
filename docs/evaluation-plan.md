## 0. 一句话结论

光交一个公开数据集的 mAP **不够**。赛题要"**仿真或实船测试结果**"。
最强提交 = **三层证据**：① 真实海事数据上的量化指标（硬分主力）+ ② 我们自己硬件的物理 demo（答辩命门）+ ③ 仿真补物理做不到的（雾）。
**精度数字靠 ①③，物理 demo 出"准确率 vs 距离"这张杀手图。**

## 2. 提交的指标（C3.md §4.1/4.2）

| 指标 | 含义 | 脚本对应 | 状态 |
|---|---|---|---|
| 检测精度 | mAP + 分类准确率 | `rfdetr_eval.py` mAP@50 / @50:95 | ✅ |
| 误检率 | 虚假目标比例（海浪杂波误判）= 1−precision | per-class FDR | ✅ |
| 漏检率 | 真实目标未检出 = 1−recall | per-class miss% + recall-by-size | ✅ |
| 实时性 | 单帧处理时间 + 硬件资源占用 | 待补：FPS/延迟 + 显存/算力（Jetson 上测）| 🔜 |
| 算法创新性 | **多模态融合** / 抗干扰 / 轻量化 | 雷达融合在此得分 | 🔜 |

**答辩提交物**：完整代码（可能查重）+ 方案说明（**建议附 git commit 记录**作工作量/创新性佐证）+ 路演 PPT + vlog（≤6min，≤3G）。
**排名**：作品排名 = **客观测评排名 + 线下评审排名 之和**，和越小越靠前（硬指标与答辩各约一半）。

## 3. 三层测试框架

### ① 量化基准（硬数字）— 真实海事数据　🔜（主线）
- 测试集：SeaShips + SMD 测试集 /（含雷达时）WaterScenes 测试集。
- **分级退化版**：用大气散射雾仿真 + 低光 + 噪声，对真实图按能见度分档。
- **雷达融合消融**：RGB vs RGB+雷达（WaterScenes 测试集）→ 定量证明融合压低雾天漏检。
- 覆盖泳池给不了的「小目标 / 远距离 / 雾天」硬指标，也是被打分的客观分。

### ② 物理硬件 demo — 我们真的 IWR6843 + 相机　⏸（主线结果后）
- 价值：唯一能证明"真多模态系统在自家硬件端到端跑起来"，答辩含金量最高。
- **场地：去真实水域**（湖/水库/海边码头），目标拉到 **20–200m**；**不要用泳池出精度**。
  - RC 船 = 移动目标；浮标 / 真实小船 = 静态目标。
  - 传感器装履带机器人/三脚架；RC 船在不同距离跑 → **"准确率 vs 距离"曲线**（杀手图，须真实距离才有意义）。
- **泳池只作**：第一次系统联调 + 上雾机/调暗灯的**可控退化演示**，不出精度数据。
- ⚠ 泳池硬伤（为什么不能当主测）：
  1. 尺度/距离不对——模型小船在近处是大目标，测不出远距小目标漏检；
  2. 毫米波看 **RCS**，3D 打印/塑料小船 RCS 极小，近距可能**直接看不到** → 雷达 demo 失败，需金属/真实 RCS 目标；
  3. 池壁强反射，**多径 ≠ 开阔海面杂波**，数据不具代表性；
  4. 池水/室内光与海面（浪、眩光、地平线、霾）**域差大**，结果难回迁。

### ③ 仿真（补物理做不到的：雾）— VRX　⏸（已搭大致环境，暂搁）
- **VRX**（Virtual RobotX，Gazebo 海事仿真）：环境已大致搭好。
- 角色：琼州海峡 <500m 的雾**物理上无法安全/可复现地造**，用仿真出 **"mAP vs 能见度"曲线**，直接对应赛题场景；也可生成配对深度（深度本就走仿真/接口）。
- **决定：先搁置**，把主线检测结果跑完再回来接 VRX。
